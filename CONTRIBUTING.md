# Contributing

Thanks for contributing.

## Before you start

1. Open an issue describing the change.
2. Wait for maintainer feedback on scope/approach.
3. Use Conventional Commits (`feat:`, `fix:`, `docs:`, etc.).

## Development setup

```bash
uv sync --all-groups
just setup
```

## Validation

```bash
just verify
```

## Pull requests

- Keep PRs focused.
- Add/update tests.
- Update docs when behavior changes.
- Ensure CI passes.
